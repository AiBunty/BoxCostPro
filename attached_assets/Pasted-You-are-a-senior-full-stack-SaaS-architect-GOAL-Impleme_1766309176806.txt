You are a senior full-stack SaaS architect.

GOAL:
Implement a STRICT multi-step onboarding flow where a user must:
1) Complete all business and pricing setup
2) Submit profile for manual admin verification
3) Get approved by admin
ONLY THEN can access Calculator, Quotes, Reports.

CURRENT PROBLEM:
- Users can access calculator immediately after login
- Setup data is incomplete
- No admin approval flow exists

EXPECTED USER FLOW:
Login →
Mandatory Setup Wizard →
Submit for Verification →
Account Locked →
Admin Approval →
Full App Access

--------------------------------------------------

Create storage methods:
- getOnboardingStatus(userId)
- updateOnboardingStep(userId, step)
- submitForVerification(userId)
- approveUser(userId)
- rejectUser(userId, reason)

Admin APIs:
- GET /api/admin/pending-verifications

- POST /api/admin/reject-user

--------------------------------------------------
4) AUTH & ACCESS MIDDLEWARE (CRITICAL)
--------------------------------------------------

Create middleware: checkOnboardingAccess

Logic:
- Fetch onboarding_status
- IF any setup step incomplete → redirect to /onboarding
- IF submitted_for_verification = true AND admin_approved = false
  → return 403 “Awaiting Admin Approval”
- IF admin_approved = false
  → block calculator, quotes, reports

Apply middleware to:
- /calculator
- /quotes
- /reports
- all pricing APIs

Create page:
client/src/pages/onboarding/index.tsx

Wizard Steps (cannot skip):
1️⃣ Business Profile + Logo Upload
2️⃣ Paper Price Settings (bulk upload + default)
3️⃣ Flute Settings
